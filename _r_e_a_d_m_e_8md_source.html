OPIC
Object Persistence In C
README.md
1 Object Persistence In C (Beta)
2 ===================================
3 
4 [![Build Status](https://travis-ci.org/dryman/opic.svg?branch=master)](https://travis-ci.org/dryman/opic)
5 
6 OPIC is a revolutionary serialization framework for C. Unlike traditional
7 approaches which walk through the in-memory objects and write it to disk, OPIC
8 itself is a memory allocator where all the objects created with it have the same
9 representation in memory and on disk. "Serializing/deserializing" is extreme
10 cheap with OPIC, because the memory can write directly to disk, and the
11 deserialization is simply a mmap syscall.
12 
13 
14 OPIC is suitable for building database indexes, key-value store, or even search
15 engines. At the moment of writing we provide a POC hash table to demonstrate how
16 easy it is to build an embedded key-value store engine.

20 SYNOPSIS
21 --------
22 
23 ```c
24 #include "opic/op_malloc.h"
25 
26 void simple_object_database(char* filename)
27 {
28  OPHeap* heap1, heap2;
29  FILE* fd;
30 
31  OPHeapNew(&heap1);
32 
33  struct S1* s1 = (struct S1*)OPMallocRaw(heap1, sizeof(struct S1));
34  struct S2* s2 = (struct S2*)OPMallocRaw(heap1, sizeof(struct S2));
35 
36  // object relationships in OPIC must convert to opref_t
37  s1->s2_ref = OPPtr2Ref(s2);
38 
39  // opref_t can convert back to pointer via OPRef2Ptr
40  struct S2* s2_ptr = OPRef2Ptr(heap1, s1->s2ref);
41 
42  // Serialize the heap to a file
43  OPHeapStorePtr(heap1, s1, 0);
44  fd = fopen(filename, "w");
45  OPHeapWrite(heap1, fd)
46  fclose(fd);
47  OPHeapDestroy(heap1);
48 
49  // Deserialize the heap and restore the objects
50  fd = fopen(filename, "r");
51  OPHeapRead(&heap2, fd);
52  fclose(fd);
53 
54  s1 = (struct S1*)OPHeapRestorePtr(heap2, 0);
55  s2 = s2_ptr = OPRef2Ptr(heap2, s1->s2_ref);
56  OPHeapDestroy(heap2);
57 }
58 ```

60 DEPENDENCY
61 ----------
62 
63 * C compiler with support of C11 atomics.
64  - gcc 4.9, gcc 5, gcc 6
65  - TODO: figure out which versions of clang support C11 atomics.
66 * [log4c (>= 1.2.4)](http://log4c.sourceforge.net)
67  - I guess 1.2.1 also works, but 1.2.4 was released since 2008. Getting
68  it on most distros shouldn't be hard.
69 * [cmocka (>= 1.0.1)](https://cmocka.org)
70  - Required for unit testing.
71 * GNU Autotools for people who want to build from head
72  - autoconf
73  - automake
74  - libtools

76 INSTALL

78 
79 ```bash
80 # For people who download the tarball release
81 ./configure
82 make
83 sudo make install
84 
85 # For people who clone from github
86 git submodule update --init --recursive
87 ./bootstrap.sh
88 ./configure
89 make

92 
93 User who runs OPIC on linux need to disable overcommit accounting. This is
94 because OPIC pre-allocates large memory in 64bit memory space.
95 
96 ```bash
97 sudo sysctl vm.overcommit_memory=1
98 ```
