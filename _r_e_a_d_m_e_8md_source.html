1 Object Persistence In C (Beta)
2 ===================================
3 
4 [![Build Status](https://travis-ci.org/dryman/opic.svg?branch=master)](https://travis-ci.org/dryman/opic)
5 
6 
7 OPIC is a revolutionary serialization framework for C. Unlike
8 traditional approaches which walk through the in-memory objects and
9 write it to disk, OPIC itself is a memory allocator where all the
10 objects created are backed by a memory mapped file. To save the
11 current snapshot of objects , simply call `OPHeapMSync()` and all the
12 changes are flushed to disk. Regular serialization may take several
13 seconds, even up to minutes to write large amont of data. In
14 contrast, OPIC only takes few milliseconds to perform both read and
15 write. For reading, `mmap()` system call only takes 0.0005 seconds to
16 map the file into memory region. The data is not loaded into memory
17 until the program access it. For writing, `msync()` is quite fast as
18 well.

20 SYNOPSIS --------
21 
22 Using OPIC we can quickly draft out a poor man's key value store.
23 OPIC provides two types of hash table: `OPHashTable` for fixed length
24 key (like `CHAR(20)` in database), and `PascalHashTable` for length
25 varying key. `PascalHashTable` provide a short string optimization
26 similar to what [C++ does][sso]. Instead of using 24 bytes for inline
27 string in hash table buckets, user can specify the size of the inline
28 buffer. If the size of the key exceeds the inline buffer, extra space
29 is allocated in OPIC to hold the key and the bucket would represent as
30 pointer to key instead of inline key.
31 
32 See the example below on how to write a mini key-value store in
33 few lines of C.
34 
35 [sso]: https://stackoverflow.com/questions/21694302/what-are-the-mechanics-of-short-string-optimization-in-libc?answertab=active#tab-top
36 
37 ```c
38 // gcc -lopic $(log4c-config --libs) opic_write.c -o opic_write
39 #include <opic/op_malloc.h>
40 #include <opic/hash/pascal_hash_table.h>
41 
42 int main(int argc, char** argv)
43 {
44  uint64_t data = 10;
45  OPHeap* heap = OPHeapOpen("myheap", O_RDWR | O_CREAT);
46  PascalHashTable* table = PHNew(heap,
47  20, // table size: 20 elements
48  0.7, // table load: 70%
49  10, // inline size for key
50  sizeof(data)); // value size
51  // store table ptr at slot 0
52  OPHeapStorePtr(heap, table, 0);
53 
54  PHInsert(table, "ABC", 4, &data); // ABC stored inline in table.
55  data++;
56  PHInsert(table, "DEF", 4, &data); // DEF stored inline in table.
57  data++;
58  PHInsert(table, "This is a long string", 22, &data);
59  // key stored as pointer to a string with length of 22 (last byte NULL).
60 
61  OPHeapClose(heap);
62  return 0;
63 }
64 
65 // gcc-6 -lopic $(log4c-config --libs) opic_read.c -o opic_read
66 #include <inttypes.h>
67 #include <opic/op_malloc.h>
68 #include <opic/hash/pascal_hash_table.h>
69 
70 int main(int argc, char** argv)
71 {
72  OPHeap* heap = OPHeapOpen("myheap", O_RDWR);
73  // restore table pointer from slot 0
74  PascalHashTable* table = OPHeapRestorePtr(heap, 0);
75 
76  uint64_t* val;
77  val = PHGet(table, "ABC", 4);
78  printf("ABC -> %" PRIu64 "\n", *val);
79  val = PHGet(table, "DEF", 4);
80  printf("DEF -> %" PRIu64 "\n", *val);
81  val = PHGet(table, "This is a long string", 22);
82  printf("This is a long string -> %" PRIu64 "\n", *val);
83  val = PHGet(table, "NAN", 4);
84  printf("Nan -> %p\n", val);
85 
86  OPHeapClose(heap);
87  return 0;
88 }
89 ```

134  - HashMultimap
