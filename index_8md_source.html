4 Object Persistence In C
5 -----------------------
6 
7 OPIC is a new approach to serialize general data structures, object types, and
8 primitive types. It's a new data stack from the ground up -- from object
9 oriented model, generics, to memory management. With everything redesigned for
10 scalability, we visioned a new distributed computing ecosystem for the 21st
11 century.
12 
13 Our key to success is to extend the memory oriented programming model to other
14 tier of storages: disk, SSD, network, or even tape. Accessing data should not be
15 limited by memory address on single machine; instead, all data types should be
16 serializable so that it can be shared across network or save for future use.
17 Having serialzability as the core design brings many benefits for free:
18 
19 * Scalability. When all the data types are serializable, the application can
20 easily scale up with multiple strategies, including sharing data between nodes,
21 off load certain data for later use, or build caches from the same serialization
22 abstraction.
23 
24 * Data persistence. Think of databases' durability. Since all the data types are
25 serializable, you can save the snapshot of the entire program state whenever you
26 want. This can make database applications way easier to build, and also make
27 debugging easier.
28 
29 * Generality for data processing applications. All the data processing systems,
30 including RDBMS, map reduce, search engines, even RPC involves are some sort of
31 special case of serialization. Why duplicates the effort for each of those
32 applications instead of having one powerful and optimized framework?
33 
34 Most of the functionalities will be implemented in C. Our end goal is to embed
35 this framework into other higher level languages like python, java, nodejs for
36 wider adoption.
